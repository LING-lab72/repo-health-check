# 用户使用手册

## 1. 启动系统

Windows 用户双击：

```text
Start-RepoHealth.bat
```

启动成功后浏览器打开：

```text
http://127.0.0.1:5174
```

## 2. 分析仓库

1. 在首页输入 GitHub 仓库 URL。
2. 点击“开始体检”。
3. 等待进度条完成。
4. 查看报告。

示例：

```text
https://github.com/vuejs/vue
```

## 3. 查看报告

报告包含：

- 健康总分。
- 等级 Badge。
- 六维雷达图。
- 语言分布。
- 各维度详情。
- AI 诊断建议。
- 问题列表。
- Badge 嵌入代码。

## 4. 导出报告

报告页支持：

- 导出 HTML。
- 导出 PDF。
- 生成分享卡片。
- 复制分享链接。
- 复制 README Badge。

## 5. 配置 AI Key

首页可配置 DeepSeek API Key。配置后：

- Key 仅保存在本地浏览器。
- 分析完成后自动使用 LLM 诊断。
- 不配置 Key 也会使用本地规则诊断。

## 6. 对比仓库

1. 点击 Dock 中的对比入口。
2. 输入仓库 A 和仓库 B。
3. 点击开始对比。
4. 查看 A / VS / B 总分、雷达图、维度差异和 PK Arena。
5. 可生成 PK 海报。

注意：A 和 B 不能是同一个仓库。

## 7. 查看排行榜

排行榜展示已分析仓库的健康分、等级、趋势和投票数。用户可以为仓库点赞。

## 8. 常见问题

### 无法连接后端

确认后端运行在：

```text
http://127.0.0.1:8002
```

并访问：

```text
http://127.0.0.1:8002/health
```

### 页面还是旧样式

关闭旧的前端终端，重新运行 `Start-RepoHealth.bat`，然后按 `Ctrl + F5` 强制刷新。

### 分析一直等待

可能是仓库过大或 GitHub 网络不稳定。可配置 `GIT_HTTP_PROXY` 或换一个较小仓库测试。
